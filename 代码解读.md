# prop.f90

该代码用于定义和处理预定义的气体和液体的热物理性质。

TYPE(THERMO_DATA_TYPE): 这部分声明了数组 THERMO_DATA 的元素类型为 THERMO_DATA_TYPE。THERMO_DATA_TYPE 是一个用户自定义的类型（在代码的其他部分定义），包含了多个字段，用于存储热物理性质的数据。

TARGET: 这个关键字表示 THERMO_DATA 数组可以作为指针的目标。也就是说，可以有指针指向 THERMO_DATA 数组的元素。

THERMO_DATA(-1:367): 这部分定义了数组 THERMO_DATA 的索引范围从 -1 到 367。Fortran 允许数组的索引范围不从 0 或 1 开始，因此这里的索引范围包括了负数 -1。

综上所述，这行代码定义了一个名为 THERMO_DATA 的数组，该数组包含 369 个元素（从 -1 到 367），每个元素的类型为 THERMO_DATA_TYPE，并且该数组可以作为指针的目标。这个数组用于存储不同物质的热物理性质数据。

MODULE THERMO_PROPS：定义一个名为 THERMO_PROPS 的模块。

- USE PRECISION_PARAMETERS：使用另一个模块 PRECISION_PARAMETERS，可能定义了精度参数。

- IMPLICIT NONE：强制显式声明所有变量，避免隐式声明。

   TYPE(THERMO_DATA_TYPE), TARGET:: THERMO_DATA(-1:367)声明了一个名为 THERMO_DATA 的数组，类型为 THERMO_DATA_TYPE，索引范围从 -1 到 367。

子程序 GET_PROP_INDEX:这个子程序用于查找给定物质名称 NAME 在 THERMO_DATA 数组中的索引 IND。

- 如果找到匹配的名称，则返回对应的索引，否则返回 -1。

- 如果 EXPLICIT_G_F 参数存在且物质的 POLY 字段为 'NASA9' 或 'NASA7'，则将 EXPLICIT_G_F 设置为 .TRUE.。

子程序 DEFINE_THERMO_PROPS:这个子程序用于定义 THERMO_DATA 数组中的热物理性质数据。

- 包含了多个物质的热物理性质数据的初始化。

# read.f90

## READ_SPEC子程序

这段代码是FDS(Fire Dynamics Simulator)软件中读取和处理物种(species)相关参数的部分。

**主要功能包括**:

读取输入文件中定义的物种参数。

设置预定义的物种混合物(如空气)。

处理简单化学反应中的燃料和产物。

设置物种的物理和化学属性,如分子量、比热容、扩散系数等。

处理凝结物种。

设置聚集物种的参数。

检查输入参数的有效性和一致性。

为后续计算准备必要的数据结构。

**主要的处理步骤包括**:

读取SPEC namelist,获取每个物种的属性。

设置预定义的物种混合物(如空气)。

处理简单化学反应中的燃料和产物,计算化学计量系数。

为每个物种分配内存并设置其属性。

处理聚集物种,设置粒径分布等参数。

检查输入的一致性,如确保反应物和产物的元素平衡。

设置物种的热力学和传输属性。

处理凝结物种,设置蒸发和凝结相关参数。

为后续计算准备必要的索引和数组。

这个子程序是FDS中处理物种和反应的核心部分之一,为后续的燃烧和传热计算奠定基础。
